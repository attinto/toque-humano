# Contribuyendo a Toque Humano

¡Gracias por tu interés en contribuir! Aquí te explicamos cómo hacerlo.

## Reportar bugs

Si encuentras un bug, abre un issue describiendo:

1. **Qué sucedió:** Describe el comportamiento incorrecto
2. **Qué esperabas:** Cuál debería haber sido el resultado
3. **Cómo reproducirlo:** Pasos específicos para replicar el problema
4. **Contexto:** Ejemplo de texto que causó el problema

## Sugerir mejoras

¿Tienes una idea para mejorar la skill?

- Abre un issue describiendo tu idea
- Explica el problema que resuelve
- Proporciona ejemplos de antes/después si es posible

## Enviar cambios

1. **Fork** el repositorio
2. **Crea una rama** para tu cambio: `git checkout -b mi-mejora`
3. **Haz tus cambios** en SKILL.md o README.md
4. **Prueba** tus cambios
5. **Commit** con un mensaje claro: `git commit -am "Describe tus cambios"`
6. **Push** a tu fork: `git push origin mi-mejora`
7. **Abre un Pull Request** describiendo tus cambios

## Directrices para contribuciones

- **Mantén la calidad:** El código y la documentación deben ser claros y correctos
- **Sé específico:** Usa ejemplos concretos antes/después
- **Respeta el estilo:** Sigue el formato de los patrones ya documentados
- **Documenta cambios:** Actualiza README.md si añades patrones nuevos
- **Sé respetuoso:** Mantén un tono profesional en todas las interacciones

## Añadiendo nuevos patrones

Si encuentras un patrón de IA que no está documentado:

1. Proporciona al menos 2 ejemplos (antes/después)
2. Explica por qué el patrón delata a la IA
3. Sugiere cómo detectarlo y corregirlo
4. Considera cómo se relaciona con los patrones existentes

## Preguntas?

Si tienes preguntas, abre un issue o contacta al mantenedor.

¡Gracias por contribuir!
